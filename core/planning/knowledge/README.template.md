# Knowledge Workspace: {{PROJECT_NAME}}

This directory stores local specification context for Zhulong native workflows.

It is intended for document-heavy development projects where implementation
depends on specs, design documents, QA records, meeting notes, test documents,
terminology, and operational notes.

## Files

| File | Purpose |
| --- | --- |
| `RAG_SOURCES.md` | Source document inventory generated by `zl-docs-scan`. |
| `DOC_RAG_STATUS.md` | Current document scan and indexing status. |
| `GLOSSARY.md` | Project terminology, including Japanese, Chinese, English, and code aliases. |
| `REQUIREMENT_TRACE.md` | Requirement-to-design-to-code-to-test trace notes. |
| `normalized/` | Local normalized text generated by `zl-docs-normalize`. |

## Rules

- Treat document RAG output as orientation until source documents are checked.
- Cite source files, sections, pages, rows, or issue IDs when possible.
- Keep private customer documents and generated indexes local unless the team
  creates a secure sharing process.
- Do not store secrets, credentials, production personal data, or API keys here.

## MVP Commands

```bash
zl-docs-scan --target {{PROJECT_ROOT}}
zl-docs-normalize --target {{PROJECT_ROOT}}
zl-docs-index --target {{PROJECT_ROOT}}
zl-docs-index --target {{PROJECT_ROOT}} --run
zl-docs-query --target {{PROJECT_ROOT}} "承認条件"
zl-docs-query --target {{PROJECT_ROOT}} --rag "承認条件"
```

`zl-docs-index` writes a document RAG backend handoff by default.
`zl-docs-index --run` executes the configured document RAG index command.
`zl-docs-query --rag` executes the configured document RAG query command.
Treat RAG answers as orientation until source documents are checked.
