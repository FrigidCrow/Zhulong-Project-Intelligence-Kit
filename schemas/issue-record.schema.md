# Issue Record Schema

Required sections:

1. Summary
2. Symptoms
3. Reproduction
4. Evidence
5. Graph / RAG Analysis
6. Source Verification
7. Root Cause
8. Impact Surface
9. Fix Plan
10. Implementation Result
11. Verification Result
12. Deployment Result
13. Rollback Plan
14. Follow-ups
15. AI-PIKit Evidence Writeback entries when evidence was recorded separately

Rules:

- No secrets or production credentials.
- Prefer links to local sanitized evidence over copied sensitive data.
- Mark no-hit, stale, noisy, or unavailable graph/RAG results explicitly.
