# Greenfield App Template

Use this template for a new product or application.

## Default Bias

- Capture decisions early.
- Keep requirements, architecture, and tests synchronized.
- Treat AI output as draft until verified.
- Add verification gates before the project becomes large.

## Recommended First Pass

1. Write product intent in `.planning/PROJECT.md`.
2. Define milestones in `.planning/ROADMAP.md`.
3. Record architecture decisions in `.planning/codebase/ARCHITECTURE.md`.
4. Configure build/test/lint commands.
5. Add issue and phase templates before implementation accelerates.
