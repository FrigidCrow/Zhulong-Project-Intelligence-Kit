发布级验证入口

表格说明每一层证明什么。详细输出从对应 CI run 的 artifact 获取。

入口覆盖范围证据位置
npm test纯函数、参数解析、状态迁移与策略判定CI 日志与测试 reporter
npm run verify:ci文档、RAG 边界、质量审计、runtime、Pages 与发布治理7 天 CI artifact
npm run verify:project-profiles文档密集型和非文档密集型核心闭环7 天 CI artifact
manifest: full-command-surface全部公开逻辑命令与 npm bin；在当前 tier 中只运行一次7 天 CI artifact
npm run verify:visual桌面、移动端、暗色模式、布局和品牌资源本地临时截图与文本摘要
npm run verify:release完整 gate、pack 清单、体积和供应链元数据Release 与 attestation

本地复跑

稳定基线是摘要，不替代命令。任何结论都必须能够在干净环境中重新生成。

npm ci --ignore-scripts
npm run verify:ci
npm run verify:release
npm pack --dry-run --json

Zhulong 只长期保存稳定结论。详细报告属于当次运行，源代码仓库不再承担临时制品仓库的职责。
